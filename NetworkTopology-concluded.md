# Network Topology — สรุป

## Topology คืออะไร

- **Topology** = รูปแบบการเชื่อมต่อ network เข้าด้วยกัน
- แต่ละแบบมี **จุดประสงค์ + ข้อดี/ข้อเสีย** แตกต่างกัน

---

## รูปแบบ Topology ⭐

### 1. Bus Network
- เชื่อมต่อด้วย **cable เส้นเดียว** (coaxial cable)
- ใช้ **T-connector** เชื่อมต่อ + **terminator** ที่ปลายสาย
- ❌ **ปัญหา**: traffic **ชนกันบ่อย**

### 2. Star Network
- คล้าย Bus แต่มี **อุปกรณ์ตรงกลาง** (hub / switch)
- เชื่อมต่อในลักษณะ **OSI Layer 2** (MAC Address)
- ❌ **ปัญหา**: traffic ชนกันบ่อย เช่นเดียวกับ Bus

### 3. Ring Network
- เชื่อม node แบบ **เครื่องต่อเครื่องไปเรื่อยๆ** เป็นวงกลม
- ตัวอย่าง: **Token Ring**
  - วางสายเหมือน Star (ใช้ MAU — Multistation Access Units)
  - แต่ทำงานแบบ Ring จริงๆ
- ✅ **ไม่มีปัญหา traffic ชนกัน** เพราะใช้ **token**:
  - เครื่องที่มี token เท่านั้นถึงจะส่งข้อมูลได้
  - ถ้าไม่มีอะไรส่ง → ส่ง token ไปเครื่องถัดไป

### 4. Mesh Network ⭐
- ต่อ node แต่ละตัว **เข้าหากันแบบ peer-to-peer**
- ถ้า node ไหนไม่ได้ต่อตรงกัน → ส่งผ่าน node อื่น
- **Full Mesh Network** = ทุก node เชื่อมต่อกันครบทุกคู่
- ❌ **ปัญหา**: ทุกครั้งที่เพิ่ม node ใหม่ ต้องลากสายเชื่อมกับ node อื่นทั้งหมด
- 📐 **สูตรจำนวนสาย**: `n(n−1)/2`
  - เช่น 5 เครื่อง → `5(5−1)/2 = 10` เส้น

### 5. Hybrid Network
- ผสม topology หลายแบบเข้าด้วยกันตามความเหมาะสม
- **ยืดหยุ่น** + ใช้งานได้ตรงตามจุดประสงค์มากกว่า

> **[เพิ่มโดย Claude]** ตารางเปรียบเทียบสรุป
>
> | Topology | ความน่าเชื่อถือ | ค่าใช้จ่าย | Single Point of Failure |
> |----------|-----------------|------------|--------------------------|
> | Bus | ต่ำ | ต่ำ | ใช่ (cable เส้นหลัก) |
> | Star | กลาง | กลาง | ใช่ (hub/switch) |
> | Ring | กลาง | กลาง | ใช่ (จุดขาด 1 จุด) |
> | Mesh | **สูงมาก** | **สูงมาก** | **ไม่** |
> | Hybrid | ขึ้นกับการออกแบบ | กลาง | ขึ้นกับการออกแบบ |

## Network Type ตามสถานที่ใช้งาน ⭐

### LAN — Local Area Network
- เครือข่ายภายใน **สถานที่เดียว**
- เข้าถึงได้ใน **domain เดียวกัน**
- เชื่อมต่อในลักษณะ **Layer 2**

### VLAN — Virtual LAN
- LAN ที่ถูก **แบ่งแยกที่ Layer 2** ด้วย **software** (ไม่ใช่ physical)
- Switch ตัวเดียวสามารถแบ่งกลุ่ม network ได้
- การข้าม VLAN → ต้องใช้ **Layer 3 (router)**
- **ประโยชน์**:
  - แบ่งส่วน server ตามการใช้งาน → ง่ายต่อการจัดการ
  - Apply policy ตาม network เดิมหรือ VLAN ก็ได้

### WAN — Wide Area Network
- **node ≥ 10** ตัว และตั้งอยู่**ห่างไกลกัน**
- ต้องลากสายเชื่อมต่อ หรือใช้ **VPN**

### MAN — Metropolitan Area Network
- เชื่อมต่อระหว่าง **LAN กับ WAN**
- พื้นที่ **ใหญ่กว่า** WAN
- ตัวอย่าง:
  - มหาวิทยาลัยเชื่อมต่อหลายวิทยาเขต
  - บริษัทที่มีหลายตึก

## BYOD (Bring Your Own Device) ⭐⭐

- การนำ **อุปกรณ์ส่วนตัว** (มือถือ, tablet, laptop) มาใช้ในองค์กร
- 🚨 **จุดอ่อน**: หากองค์กรไม่แยกการใช้งานระหว่างเครื่ององค์กรกับเครื่องส่วนตัว
- เครื่ององค์กร → เข้มงวดด้าน security
- เครื่องส่วนตัว → security ต่ำกว่ามาก → **เป็นช่องโจมตี**

### ภัยจาก BYOD

#### 1) เข้าถึง Network องค์กรได้ง่ายขึ้น
- เครื่องส่วนตัวในองค์กรที่ไม่แยกการใช้งาน → ผู้โจมตีเข้า network ได้ง่าย
- ปัญหาที่พบบ่อย:
  - **ACL (Access Control List) หละหลวม**
  - WFH ช่วง COVID-19 → องค์กรอนุญาต BYOD มากขึ้น
- **แนวทางแก้**:
  - แยก network ของ BYOD จาก network องค์กรอย่างชัดเจน
  - VPN ต้องแยกสิทธิ์ตามงานที่รับผิดชอบเท่านั้น

#### 2) ข้อมูลองค์กรรั่วไหลง่ายขึ้น
- เข้า mail องค์กรจากเครื่องส่วนตัวถือเป็นเรื่องปกติ
- บันทึกเอกสารสำคัญในเครื่องส่วนตัว → จุดเสี่ยงข้อมูลรั่ว
- **แนวทาง**: เครื่องผู้บริหารต้องคุมการเข้าถึง + การติดตั้ง app อย่างเข้มงวด

## Conclusion

### Topology + Network Type
- เลือก **Topology** ตามจุดประสงค์ + ข้อจำกัดในการเชื่อมต่อ
- เลือก **Network Type** ตามสถานที่ตั้งและขนาดของเครือข่าย

### BYOD — แนวทางสำคัญ
- ✅ **แยกวง network**: ใช้งานทั่วไป (เช่น internet) กับ network องค์กรสำคัญ
- ✅ **เข้าถึง network สำคัญ** เฉพาะเครื่องที่อนุญาต (เช่น เครื่ององค์กร)
- ✅ **เข้าถึง mail / เอกสารสำคัญ** เฉพาะเครื่องที่มีการป้องกันดี
- 🎯 **เป้าหมาย**: ลดความเสี่ยงจาก BYOD โดยควบคุมการเข้าถึง network และเอกสารสำคัญ
