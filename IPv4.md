IPv4
อินเตอร์เน็ตคือการนำอุปกรณ์หลายๆอย่างทำงานร่วมกัน ซึ่งประกอบไปด้วย
- คอมพิวเตอร์ (Computing Devices)
- อุปกรณ์เครือข่าย เช่น  router, switch, hub เป็นต้น
นอกจากนี้อาจจะรวมอุปกรณ์ต่างๆ ที่มีการประมวลผลในตัวเอง เช่น Smart Phone, Internet of Thing(IoT) อุปกรณ์ทั้งหลายเหล่านี้มักถูกเรียกว่า โฮสต์ (Host) หรือ ระบบปลายทาง (End System, Client)
เครื่องคอมพิวเตอร์ทั้งหลายถูกเชื่อมต่อเข้าด้วยกันเป็นจำนวนหลายล้านเครื่อง จะทำการติดต่อโดยผ่านลิงค์การสื่อสาร (Communication Link) ซึ่งจะแบ่งเป็นลิงค์แบบมีสาย เช่น ใยแก้วนำแสง(Fiber) สายทองแดง หรือ สายโคแอ็กเชี่ยล เป็นต้น  และแบบไร้สาย เช่น คลื่นวิทยุ หรือ คลื่นไมโครเวฟ ลักษณะของลิงค์การสื่อสารที่แตกต่างกันจะมีความเร็วในการส่งข้อมูลที่ต่างกัน  ความเร็วในการสื่อสารข้อมูล (Transmission rate)  จะหมายถึง แบนด์วิทธ์ (Bandwidth) ซึ่งมีหน่วยคือ บิทต่อวินาที ( bit per second, bps)

เร้าเตอร์ (Router)
เร้าเตอร์จะเป็นตัวทำหน้าที่ในการส่ง (Forwarding) แพ็คเก็ตข้อมูลที่เดินทางจากโฮสต์ ไปยังโฮสต์ต่างๆ โดยเร้าเตอร์จะทำหน้าที่ในการหาเส้นทางและเลือกเส้นทางที่จะทำการส่งผ่านแพ็คเก็ตไปยังปลายทางให้ ข้อมูลที่จะถูกส่งลงไปในระบบเครือข่ายนั้นจะถูกแบ่งออกเป็นข้อมูลกลุ่มเล็กๆ เรียงต่อกัน ข้อมูลกลุ่มเล็กๆนี้จะเรียกว่า แพ็คเก็ต (Packet) ทั้งนี้การส่ง packet จะส่งด้วย protocol ต่างๆ

โปรโตคอล (Protocol)
โปรโตคอลคือการกำหนดข้อตกลงสำหรับการสื่อสารระหว่างจุดใดๆ บนระบบเครือข่าย หรือ ถ้าจะกล่าวอย่างละเอียดคือ
โปรโตคอล ทำหน้าที่กำหนด รูปแบบ และลำดับของข้อความที่จะใช้ในการแลกเปลี่ยนระหว่างจุด 2 จุดหรือมากกว่า รวมไปถึงการกระทำต่างๆที่เกิดขึ้นระหว่างการรับและส่งข้อความ
ตัวอย่างเช่น

HTTP เป็นข้อตกลงในการสื่อสารในการแลกเปลี่ยนข้อมูลระหว่าง web server กับ client (โดยปกติจะใช้ browser ในการติดต่อสื่อสาร)
SMTP เป็นข้อตกลงในการสื่อสารรับส่ง email
การทำงานของโปรโตคอลบนระบบเครือข่ายเปรียบเสมือนกับการสื่อสารของมนุษย์นั่นเอง

IPv4
IP (Internet Protocol Address) มีหน้าที่คือเป็นหลายเลขที่ใช้ในระบบเครือข่าย เป็นหมายเลขประจำเครื่องคอมพิวเตอร์ หรือจะเรียกว่าเป็นบ้านเลขที่ของคอมพิวเตอร์ในกรณีที่เราเชื่อมต่อกับระบบเครือข่าย เพือให้เครื่องคอมพิวเตอร์หรืออุปกรณ์ต่างๆ บนระบบเครือข่ายรู้จักกัน  สามารถสื่อสารกันได้ และส่งข้อมูลให้กับเครื่องได้อย่างถูกต้อง
โดยปกติ IP นั้นจะเป็น IPv4 โดยจะเป็นหมายเลขที่มีทั้งหมด 32 บิตแต่ละช่วงจะถูกขั้นด้วย “.” แบ่งเป็นขุดละ 8 บิต โดยตัวเลข 8 นี้ก็จะมีค่าตั้งแต่ 0 – 255 ดังนั้น IPv4 จึงมีหมายเลขได้ตั่งแต่ 0.0.0.0 ถึง 255.255.255.255 แต่จะมีบางหมายเลขที่ถูกเก็บไว้ใช้งานเฉพาะ
IPv4 ทั้งหมดถูกแบ่งออกเป็น Class ชนิดต่างๆตามจุดประสงค์การใช้งาน ประกอบไปด้วยดังต่อไปนี้
- Class A คือ 1.0.0.1 - 127.255.255.254
- Class B คือ 128.0.0.1 - 191.255.255.254
- Class C คือ 192.0.1.1 - 223.255.254.254
- Class D คือ 224.0.0.0 - 239.255.255.255 ใช้สำหรับงาน multicast
- Class E คือ 240.0.0.0 - 255.255.255.254 ถูกสำรองไว้ ยังไม่มีการใช้งาน

โดย IP Address นั้นถูกจัดการโดย Internet Assigned Numbers Authority (IANA) และ regional Internet registries (RIR) เพื่อแจกจ่าย IP address ให้กับ user หรือ  Internet service providers (ISP) ต่างๆ เพื่อนำไปใช้งานโดยไม่ซ้ำกัน ซึ่ง ISP จะนำ IP ที่ได้รับมานั้นแจกจ่ายให้กับ user ทั้งหลายอีกที ซึ่งอาจจะเป็นทั้งแบบ static IP (ตายตัวไม่เปลี่ยนแปลง) หรือ dynamic IP (เปลี่ยนทุกครั้งที่ connection ใหม่) โดยขึ้นอยู่กับการจัดสรรและการจัดการของทาง ISP นั้นๆ

สำหรับไอพีในช่วง 127.0.0.0 ถึง 127.255.255.255 นั้นจะใช้สำหรับเพื่อทดสอบ ซึ่งปกติก็คือ IP ของ localhost หรือก็คือเครื่องของเรานั่นเอง โดย IP Add ทั้งหมดจะมี address ทั้งหมดจำนวน 4,294,967,296 IP address (232)

Subnet Mask
Subnetwork หรือก็คือ subnet จะเป็นส่วนการแบ่ง IP Address Network โดยการแบ่ง network ให้กลายเป็นมากกว่า 2 กลุ่ม จะเรียกว่า subnetting.
กลุ่ม computer ที่อยู่ภายใต้ subnet เดียวกัน ย่อมมี bit-group ใน IP address เหมือนกัน เช่น 192.168.100.1 กับ 192.168.100.2  ทำให้เราสามารถแบ่งส่วนของ IP address ออกเป็น 2 ส่วนคือ
- Network prefix หรือ Routing prefix
- Host Number
Network prefix เขียนในลักษณะ IP address แรกของ subnet เช่น 192.168.100.0 แล้วตามด้วย slash (/) จามด้วย bit length ของ subnet นั้นๆ เช่น /24 รวมกันจะออกมาเป็น 192.168.100.0/24 (prefix IPv4) โดย 24 มาจาก Network IP เป็น 24 bit แรกและ host 8 bit จะเป็น bit ที่เหลือนั่นคือ 32 - 24 = 8 bit ซึ่งก็คือ 192.168.100 ตัวเลข 3 ชุดด้านหน้ารวมกันทั้งหมดเท่ากับ 24 bit  ส่วนชุดสุดท้ายคือ 0-255 นั่นคือ host number มีค่าเท่ากับ 8 bit นั่นเอง ทั้งนี้เราสามารถเขียนได้อีกแบบออกมาเป็น 255.255.255.0 ซึ่งก็คือ 24bit ด้านหน้า ถูก set เป็น 1 นั่นเอง

ตัวอย่างการคำนวณ subnet
การคำนวณ subnet โดยอาศัยข้อมูลจากตารางอ้างอิงในการอธิบาย

 	Dot-decimal notation	Binary form
Address	192.168.100.0	11000000.10101000.01100100 .00000000
Netmask	255.255.255.0 = 24	11111111.11111111.11111111 .00000000
Wildcard	0.0.0.255	00000000.00000000.00000000 .11111111
Network	192.168.100.0/24	11000000.10101000.01100100 .00000000 (Class C)
Broadcast	192.168.100.255	11000000.10101000.01100100 .11111111
HostMin	192.168.100.1	11000000.10101000.01100100 .00000001
HostMax	192.168.100.254	11000000.10101000.01100100 .11111110
จากตารางจะเห็นว่า Netmask นั้นจะขึ้นด้วยชุดของ 1 จำนวน 24 ตัวหรือ 24 bit จากนั้นตามด้วยชุดของ 0 อีก 8 bit รวมกันทั้งสิ้น 32 bit ซึ่งจำนวนของ 0 ที่ต่อท้ายคือจำนวนของ host ที่จะมีได้ใน subnet นี้นั้นเอง จากตารางจะเห็นว่า HostMin คือ 1 เครื่อง และสามารถมีเครื่องได้สูงที่สุด (HostMax) คือ 254 เครื่อง

การทำ Subnetting
การทำ subnetting คือการแบ่ง network mask จากเดิมของ subnet mask ให้แบ่งเป็นกลุ่มมากขึ้น โดยทำการเปลี่ยน bit ของ network mask ในส่วนของ Host part เพิ่มเติม โดยจากตัวอย่างเป็น

 	Dot-decimal notation	Binary form
Address	192.168.100.0	11000000.10101000.01100100 .00000000
ในที่นี้จะกำหนด subnet mask เป็น 2 subnet ย่อยจาก 255.255.255.0 กลายเป็น 255.255.255.192(/26) ไปเป็น

 	Dot-decimal notation	Binary form
Netmask	255.255.255.192	11111111.11111111.11111111 .11000000
จะได้ออกมาเป็น IP 2 วงคือ

 	Dot-decimal notation	Binary form
Address1	192.168.100.64/26	11000000.10101000.01100100 .01000000
Address2	192.168.100.128/26	11000000.10101000.01100100 .10000000
Host length ของแต่ละ subnet จะมีรายละเอียดตามด้านล่าง ทั้งนี้ IP สุดท้ายของแต่ละ subnet จะถูกจองไว้สำหรับเป็น broadcast IP

11000000.10101000.01100100.01000001 - 11000000.10101000.01100100.01111110
11000000.10101000.01100100.10000001 - 11000000.10101000.01100100.10111110
