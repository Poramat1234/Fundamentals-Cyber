# IPv4 — สรุป

## ภาพรวมของอินเทอร์เน็ต

- **อินเทอร์เน็ต** = อุปกรณ์หลายชนิดทำงานร่วมกัน:
  - คอมพิวเตอร์ (Computing Devices)
  - อุปกรณ์เครือข่าย: router, switch, hub
  - อุปกรณ์ที่มีการประมวลผลในตัว: smartphone, IoT
- อุปกรณ์เหล่านี้เรียกรวมว่า **Host / End System / Client**
- เชื่อมต่อกันผ่าน **Communication Link**
  - **มีสาย**: Fiber, สายทองแดง, Coaxial
  - **ไร้สาย**: คลื่นวิทยุ, ไมโครเวฟ
- **Transmission rate (Bandwidth)** วัดเป็น **bps** (bit per second)

## Router

- ทำหน้าที่ **forward packet** จาก host หนึ่งไปยัง host ปลายทาง
- **หาเส้นทาง** และ **เลือกเส้นทาง** ที่ดีที่สุด
- ข้อมูลจะถูกแบ่งเป็นกลุ่มเล็กๆ เรียกว่า **Packet**
- ส่ง packet ผ่าน **Protocol** ต่างๆ

## Protocol

- ข้อตกลงสำหรับการสื่อสารระหว่างจุดต่างๆ บนเครือข่าย
- กำหนด **รูปแบบ + ลำดับ** ของข้อความที่แลกเปลี่ยนกัน
- ตัวอย่าง:
  - **HTTP** — แลกเปลี่ยนข้อมูลระหว่าง web server กับ client
  - **SMTP** — รับส่งอีเมล

## IPv4 — แก่นหลัก ⭐

- **IP (Internet Protocol Address)** = หมายเลขประจำเครื่องในเครือข่าย (เปรียบเหมือน "บ้านเลขที่")
- **IPv4 = 32 bits** แบ่งเป็น 4 ชุด ชุดละ 8 bits คั่นด้วย `.`
- ค่าแต่ละชุด: **0–255**
- ช่วงทั้งหมด: `0.0.0.0` – `255.255.255.255`
- จำนวน address ทั้งหมด: **2³² = 4,294,967,296**

### Class ของ IPv4

| Class | ช่วง | การใช้งาน |
|-------|------|-----------|
| **A** | 1.0.0.1 – 127.255.255.254 | เครือข่ายขนาดใหญ่มาก |
| **B** | 128.0.0.1 – 191.255.255.254 | เครือข่ายขนาดกลาง |
| **C** | 192.0.1.1 – 223.255.254.254 | เครือข่ายขนาดเล็ก |
| **D** | 224.0.0.0 – 239.255.255.255 | **Multicast** |
| **E** | 240.0.0.0 – 255.255.255.254 | สำรอง ยังไม่ใช้งาน |

### การจัดสรร IP

- **IANA** (Internet Assigned Numbers Authority) + **RIR** (regional Internet registries) → แจกจ่ายให้ user / **ISP**
- ISP แจกต่อให้ผู้ใช้เป็น:
  - **Static IP** — ตายตัว
  - **Dynamic IP** — เปลี่ยนทุกครั้งที่ connection ใหม่

### IP พิเศษ

- `127.0.0.0` – `127.255.255.255` → **Loopback / localhost** (ทดสอบ, เครื่องเรา)

## Subnet Mask & Subnetting ⭐

- **Subnet** = การแบ่ง IP Address Network ออกเป็นกลุ่มย่อย
- **Subnetting** = การแบ่งเป็นมากกว่า 2 กลุ่ม
- เครื่องใน subnet เดียวกัน → มี **bit-group เดียวกัน** ใน IP

### โครงสร้าง IP

แบ่งออกเป็น 2 ส่วน:
1. **Network Prefix (Routing Prefix)**
2. **Host Number**

### Notation

- รูปแบบ: `<network>/<prefix-length>` เช่น `192.168.100.0/24`
- `/24` = network 24 bit แรก, host 8 bit ที่เหลือ (32 − 24 = 8)
- เขียนเป็น subnet mask ก็คือ `255.255.255.0` (24 bit แรกเป็น 1)

### ตัวอย่างคำนวณ Subnet — `192.168.100.0/24`

| ค่า | Dot-decimal | Binary |
|-----|-------------|--------|
| Address | 192.168.100.0 | `11000000.10101000.01100100.00000000` |
| Netmask | 255.255.255.0 (/24) | `11111111.11111111.11111111.00000000` |
| Wildcard | 0.0.0.255 | `00000000.00000000.00000000.11111111` |
| Network | 192.168.100.0/24 | (Class C) |
| **Broadcast** | 192.168.100.255 | `...11111111` |
| **HostMin** | 192.168.100.1 | `...00000001` |
| **HostMax** | 192.168.100.254 | `...11111110` |

- จำนวน 0 ที่ต่อท้าย = จำนวน host ที่ใช้งานได้ → **/24 รองรับ 254 host**

### การแบ่ง Subnet เพิ่มเติม — เปลี่ยนจาก /24 → /26

- Netmask `/26` = `255.255.255.192` = `11111111.11111111.11111111.11000000`
- ได้ 2 subnet ย่อย:
  - `192.168.100.64/26`
  - `192.168.100.128/26`
- IP ตัวสุดท้ายของแต่ละ subnet = **broadcast** (ใช้งานไม่ได้)

### ตาราง Subnet ตาม Prefix Size ⭐

| Prefix | Subnet mask | Subnets | Hosts/subnet | Total hosts |
|--------|-------------|---------|--------------|-------------|
| /24 | 255.255.255.0 | 1 | 254 | 254 |
| /25 | 255.255.255.128 | 2 | 126 | 252 |
| /26 | 255.255.255.192 | 4 | 62 | 248 |
| /27 | 255.255.255.224 | 8 | 30 | 240 |
| /28 | 255.255.255.240 | 16 | 14 | 224 |
| /29 | 255.255.255.248 | 32 | 6 | 192 |
| /30 | 255.255.255.252 | 64 | 2 | 128 |
| /31 | 255.255.255.254 | 128 | 0* | — |

\* `/31` ใช้งานไม่ได้เพราะถูกเก็บไว้เป็น Network ID + Broadcast ID

## Private vs Public IP ⭐

- **Public IP** → ใช้สื่อสารบน Internet
- **Private IP** → ใช้ใน local network ภายในองค์กร

### ช่วง Private IP

| Block | ช่วง | จำนวน address |
|-------|------|---------------|
| 24-bit (/8, 1×A) | 10.0.0.0 – 10.255.255.255 | 16,777,216 |
| 20-bit (/12, 16×B) | 172.16.0.0 – 172.31.255.255 | 1,048,576 |
| 16-bit (/16, 256×C) | 192.168.0.0 – 192.168.255.255 | 65,536 |

> **[เพิ่มโดย Claude]** Private IP เชื่อมต่อ Internet ได้ผ่าน **NAT (Network Address Translation)** ซึ่ง router จะแปลง private IP เป็น public IP ขาออก ทำให้หลายเครื่องใน LAN ใช้ public IP เดียวร่วมกันได้ — ช่วยบรรเทาปัญหา IPv4 หมด

---

## สรุปสำคัญ

- **IPv4 = บ้านเลขที่** ของเครื่องบนเครือข่าย (32 bit)
- แบ่ง Class **A–E** ตามจุดประสงค์
- ใช้ **Subnet Mask** แบ่งวง network ออกเป็นกลุ่มย่อยเพื่อบริหารจัดการ
- มี **Private IP** สำหรับใช้ภายใน + **Public IP** สำหรับ Internet
