Network Topology
Topology คือการเชื่อมต่อ network เข้าด้วยกัน โดยการเชื่อมต่อแต่ละแบบก็มีจุดประสงค์ในการเชื่อมต่อแตกต่างกัน รวมถึงข้อดีข้อเสียก็แตกต่างกันอีกด้วย

Bus Network
Bus network เป็นการเชื่อมต่อที่มีเส้น cable เป็นตัวเชื่อมเส้นเดียว bus network ใช้สาย coaxial cable ซึ่งมีตัวเชื่อมต่อเป็น T-connector และมีตัวสวมในต้นและปลายของสายเรียกว่า terminator ปัญหาของการเชื่อมต่อแบบนี้คือเกิดการชนกันของ traffic บ่อย


Bus Network
Star Network
Star network จะคล้ายๆกับ bus network แตกต่างที่มีตัวเชื่อมทั้งหมดอยู่ ซึ่งอาจเป็นได้ทั้ง hub, switch โดยจะเชื่อมต่อในลักษณะของ OSI Layer 2 นั่นคือ MAC Address ซึ่งปัญหาไม่แตกต่างจาก bus network นั่นคือเกิดการชนกันของ traffic บ่อย


Star Network
Ring Network
Ring network นั้นจะคล้ายๆกับ bus network อีกเช่นเคย โดยมีการเชื่อม node ต่างๆเข้าด้วยกันผ่านเส้น network เดียว แต่จะเป็นการเชื่อมแบบเครื่องต่อเครื่องไปเรื่อยๆ หนึ่งใน network ที่เป็น ring network นั่นคือ token ring. ในระบบ token ring network, ระบบจะถูกวางสายเหมือนกับ star, โดยใช้เครื่องหลายๆเครื่อง (multistation access units (MAUs)). แต่ไม่ได้มีการทำงานในลักษณะ start แต่อย่างใด 


Ring Network
ใน Ring network นั้นจะไม่มีปัญหา traffic ชนกันแล้วเพราะเชื่อมต่อหากันโดยใช้ token. โดยระบบที่มี token เท่านั้นถึงจะได้ส่งข้อมูล หากไม่มีอะไรจะส่ง token ก็จะถูกส่งไปยังเครื่องถัดไป

Mesh Network
Mesh network เป็นการต่อ node แต่ละ node เข้าด้วยกัน โดยจะทำให้ดูซับซ้อนมากขึ้น แต่ก็ทำให้เชื่อมต่อกันโดยตรงแบบ peer to peer หากเครื่องใดไม่เชื่อมต่อกันก็ต้องส่งผ่านช่องทางของเครื่องอื่น


Mesh Network
โดยการเชื่อมต่อ Mesh Network เข้าด้วยกัน จะเรียกว่า “Full Mesh Network” 


Full Mesh Network
ปัญหาที่เกิดขึ้นของ Mesh Network คือ เมื่อมีการนำ node ใหม่เข้าไปต่อใน network เราต้องเชื่อมต่อ node นั้นกับ node อื่นทั้งหมด ทำให้สาย connection ที่ใช้นั้นเพิ่มอย่างทวีคูณอย่างมาก โดยจำนวนของสายที่ใช้จะมีค่าเท่ากับ n(n-1)/2 เช่น

หากมี 5 เครื่อง, จะใช้ทั้งหมด 5(5–1)/2​ เท่ากับ 10 เส้นนั่นเอง

Hybrid network
Hybrid network เป็นการเลือกการเชื่อมต่อตามความเหมาะสมของการเชื่อมต่อ เป็นการเชื่อมต่อหลายๆ network เข้าหากัน แต่แต่ละ network อาจจะมีลักษณะที่แตกต่างกันออกไปก็ได้เช่นกัน ดังนั้นจะทำให้ยืดหยุ่นและใช้งานได้อย่างตรงตามความเหมาะสมมากกว่า

Network Type
Network type คือประเภทของ network ตามสถานที่ใช้งาน ทั้งนี้ประกอบไปด้วย

Local Area Network (LAN)
เป็นเครือข่ายที่อยู่ภายในสถานที่ใดสถานที่หนึ่ง ซึ่งสามารถถูกเข้าถึงได้ใน domain เดียวกัน มักเชื่อมต่อในลักษณะของ Layer2 

Virtual Local Area Network (VLAN)
VLAN เป็น LAN ที่ถูกแบ่งแยกออกจากกันที่ layer2 ซึ่งจัดการโดย software มากกว่าที่จะเป็น physical นั่นหมายความว่า switch บางตัวสามารถที่จะแบ่งกลุ่มของ network ออกจากกันได้เป็นกลุ่มๆ ด้วยการใช้งาน VLAN ที่แตกต่างกันออกไป การที่จะเชื่อมต่อจาก VLAN หนึ่งไปยังอีก VLAN หนึ่งได้ ต้องเชื่อมต่อในลักษณะ layer3 โดยใช้ router

การแบ่ง VLAN นั้นอาจใช้เพื่อแบ่งส่วนของ server ที่ใช้งานที่แตกต่างกันเพื่อให้ง่ายต่อการจัดการ ทั้งนี้การ apply policy อาจแบ่งไปตาม network เดิม หรืออาจจะ apply ตาม VLAN ก็สามารถทำได้เช่นเดียวกัน

Wide Area Network (WAN)
WAN network เป็นลักษณะที่จำนวนของเครื่องที่เชื่อมต่อ (node) มากกว่า 10 node ขึ้นไปและมีสถานที่ที่ตั้งที่ห่างไกลกัน ซึ่งหลายๆครั้งต้องมีการรากสายเชื่อมต่อหากัน หรืออาจจะเชื่อมต่อหากันในลักษณะ VPN ก็เป็นไปได้เช่นเดียวกัน


Metropolitan Area Network (MAN)
MAN เป็นการเชื่อมต่อของ LAN และ WAN ซึ่งอาจจะเจอในลักษณะของมหาวิทยาลัยที่เชื่อมต่อกับ วิทยาเขตต่างๆเข้าหากัน หรืออาจจะเป็นบริษัทที่มีตึกหลายๆตึกก็เป็นได้เช่นกัน ซึ่งพื้นที่การเชื่อมต่อของ MAN นั้นจะใหญ่กว่าเมื่อเทียบกับ WAN


BYOD
Bring Your Own Device (BYOD) เป็นการนำอุปกรณ์ส่วนตัวใดๆ ไม่ว่าจะเป็นมือถือ, tablet, laptop นำมาใช้ในองค์กร ซึ่งหากองค์กรไม่ได้แยกการใช้งานระหว่างเครื่องของภายในบริษัทกับเครื่องส่วนตัวอาจเป็นจุดอ่อนขององค์กรได้ แน่นอนว่าโดยปกติแล้วเครื่องขององค์กรมักถูกติดตั้ง Application อย่างรัดกุม รวมถึงกฏข้อบังคับการใช้งานต่างๆที่เข้มงวด ดังนั้นการโจมตีเข้าไปยังในเครื่องขององค์กรนั้นอาจจะทำได้ยาก แต่หากมองไปที่เครื่องมือส่วนตัวซึ่งมีความเข้มงวดในการติดตั้งน้อยกว่ามาก ในจุดนี้ทำให้ผู้โจมตีมองเห็นถึงประโยชน์และอาจใช้เป็นช่องทางการโจมตีและยึดเครื่องของเป้าหมายนั่นเอง

 

ภัยที่มากับ BYOD
1) การเข้าถึง Network ขององค์กรเป้าหมายได้ง่ายขึ้น
เมื่อเป้าหมายนำเครื่องไปใช้ภายในองค์กรที่ไม่ได้แยกการใช้งาน ผู้โจมตีจะสามารถเข้าถึง network ได้ง่ายมากขึ้น เพราะหลายๆองค์กรคิดว่าเครื่องขององค์กรนั้นเข้มงวดมาก ดังนั้นจึงไม่ระมัดระวังการใช้งานในส่วนของเครือข่าย การหละหลวมของการกำหนดสิทธิ์การเข้าถึงเครือข่ายภายใน (Access Control List:ACL) นั้นทำให้การโจมตีภายในองค์กรนั้นทำได้ง่ายขึ้น อีกกรณีที่เป็นไปได้นั่นคือ ณ ปัจจุบันองค์กรหลายๆองค์กรปรับเปลี่ยนองค์กรให้ทำงานในลักษณะ Work From Home สามารถให้พนักงานทำงานจากที่บ้านได้ ในหลายๆองค์กรก็ปรับ policy ให้สามารถใช้เครื่องคอมพิวเตอร์ของส่วนตัวนำมาใช้เพื่อทำงานขององค์กรได้เนื่องด้วยการแพร่ของ COVID-19 จึงทำให้เป็นช่องทางให้ผู้โจมตีใช้เป็นการเข้าถึงเครือข่ายภายในองค์กรได้ง่ายมากขึ้น ดังนั้นการแยกการใช้งานเครือข่ายของเครื่ององค์กรและเครื่องของใช้ส่วนตัวควรจะแยกกันอย่างชัดเจน การออกแบบ network ควรทำเครือข่ายเฉพาะของ BYOD โดยเฉพาะใช้งาน เพื่อให้ไม่ปะปนกับการใช้งานของอุปกรณ์ขององค์กร หากแม้องค์กรยินยอมให้ใช้งาน VPN Network ในการเชื่อมต่อเครือข่ายบริษัท ก็ควรจำเป็นต้องแยกสิทธิ์การใช้งานเฉพาะเจาะจงตามงานที่พนักงานคนนั้นรับผิดชอบเท่านั้น ไม่ควรจะให้สิทธิพิเศษต่อการทำงานแต่อย่างใด

2) ข้อมูลขององค์กรถูกเข้าใช้งานได้ง่ายขึ้นจากการใช้งาน BYOD 
การใช้งานเครื่องส่วนตัวในการเข้าถึง mail ขององค์กรถือเป็นเรื่องปกติไปเสียแล้ว หรือการใช้งานเครื่องส่วนตัวเพื่อเข้าส่วนทดสอบหรือแม้แต่การบันทึกเอกสารสำคัญภายในเครื่องส่วนตัวนั้นถือเป็นจุดที่ทำให้ถูกโจมตีและข้อมูลรั่วไหลได้อีกทางหนึ่ง ดังนั้นจะมีในบางองค์กรที่เครื่องของผู้บริหารจะต้องถูกควบคุมการเข้าถึงและการติดตั้ง Application อย่างเข้มงวด เพื่อให้เอกสารสำคัญขององค์กรไม่หลุดออกไปนั่นเอง

Conclusion
จากหัวข้อนี้ได้จะได้เรียนรู้การเชื่อมต่อแต่ละ network เข้าด้วยกันในรูปแบบ (Topology) และลักษณะของ network ตามสถานที่ตั้งต่างๆ

สำหรับในส่วนของ BYOD นั้นในทุกๆองค์กรควรมีแยกวง network สำหรับการใช้งานโดยทั่วไป เช่นการใช้งาน internet เป็นต้น กับ network ภายในองค์กรที่สำคัญ ซึ่งการเข้าถึงเครือข่ายสำคัญดังกล่าว ควรจะถูกเข้าถึงได้โดยเครื่องที่อนุญาต เช่น เครื่องขององค์กรเท่านั้น เป็นต้น และการเข้าถึงระบบ mail หรือเอกสารสำคัญควรถูกเข้าถึงได้เฉพาะเครื่องที่มีการป้องกันเป็นอย่างดี เพื่อให้ไม่เกิดปัญหาในการควบคุมการเข้าถึง network ขององค์กร และเอกสารสำคัญหรือ email ขององ์คกร เพื่อลดความเสี่ยงที่อาจจะเกิดขึ้นจากการใช้งาน BYOD
